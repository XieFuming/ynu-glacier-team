# 团队资料补充与更新方法

网站内容主要维护在 `data/*.json`。JSON 文件必须保持英文逗号、英文双引号和合法数组格式；修改后建议先本地预览。

## 修改团队基本信息

文件：`data/site.json`

可更新内容：

- `tagline`：首页一句话介绍
- `intro`：团队简介
- `stats`：首页统计数据
- `researchAreas`：研究方向
- `contact`：邮箱、地址、依托单位、招生方向

示例：

```json
{
  "label": "长期观测区",
  "value": "6"
}
```

## 新增团队成员

文件：`data/members.json`

在数组中增加一段：

```json
{
  "name": "真实姓名",
  "initials": "姓名首字母",
  "role": "教授 / 博士生导师",
  "group": "教师",
  "bio": "一句话简介，建议 60-120 字。",
  "tags": ["冰川变化", "遥感", "水文模型"]
}
```

`group` 会自动生成筛选按钮，常用值可设为：`教师`、`博士后`、`研究生`、`合作伙伴`、`毕业生`。

## 新增论文

文件：`data/publications.json`

在数组中增加一段：

```json
{
  "year": 2026,
  "title": "论文题名",
  "authors": "作者 A, 作者 B, 作者 C",
  "journal": "期刊名, 卷(期), 页码",
  "doi": "10.xxxx/xxxxx",
  "url": "https://example.com/paper",
  "keywords": ["冰川", "气候变化"]
}
```

说明：

- 没有 DOI 时，`doi` 保持空字符串 `""`。
- 没有全文链接时，`url` 保持空字符串 `""`。
- 年份会自动生成筛选按钮。

## 新增科研项目

文件：`data/projects.json`

```json
{
  "year": 2026,
  "period": "2026-2030",
  "level": "国家自然科学基金",
  "title": "项目名称",
  "description": "负责人、批准号、研究目标或项目简介。"
}
```

## 新增奖励荣誉

文件：`data/awards.json`

```json
{
  "year": 2026,
  "title": "奖励或荣誉名称",
  "organization": "授奖单位"
}
```

## 新增新闻动态

文件：`data/news.json`

```json
{
  "date": "2026-05-23",
  "title": "新闻标题",
  "summary": "新闻摘要，建议 60-120 字。"
}
```

日期格式固定为 `YYYY-MM-DD`。

## GitHub 网页端更新方法

1. 打开 GitHub 仓库。
2. 进入要修改的文件，例如 `data/news.json`。
3. 点击右上角铅笔按钮编辑。
4. 修改内容后点击 `Commit changes`。
5. 等待 GitHub Pages 自动部署，一般几分钟内生效。

## 建议准备的真实材料清单

- 团队正式名称、英文名称、依托学院或实验室
- 团队负责人简介、邮箱、个人主页
- 每位成员姓名、职称/年级、研究方向、照片或头像
- 近五年代表论文，含 DOI 或期刊链接
- 在研项目，含项目来源、批准号、周期和负责人
- 获奖荣誉，含年份和授奖单位
- 新闻动态，含日期、标题、摘要和照片
- 学生招生说明、合作方向和联系邮箱
